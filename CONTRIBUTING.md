# Contributing to HCP Reference

Thank you for your interest in contributing to the Humanitarian Connection Protocol (HCP) Reference implementation.

HCP is an open-source project whose mission is to provide a standardized, interoperable foundation for humanitarian information exchange.

We welcome contributions from developers, researchers, humanitarian organizations and anyone interested in improving humanitarian interoperability.

---

# Before You Start

Please read:

- README.md
- docs/architecture.md
- HCP Specification repository

Understanding the protocol architecture before contributing helps maintain consistency across the project.

---

# Ways to Contribute

You can contribute by:

- Fixing bugs
- Improving documentation
- Writing tests
- Reviewing code
- Implementing protocol features
- Improving performance
- Reporting issues
- Proposing enhancements

---

# Development Principles

Every contribution should follow these principles:

- Protocol First
- Simplicity
- Interoperability
- Security
- Backward Compatibility
- Clear Documentation

The protocol is always more important than any individual client implementation.

---

# Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Make focused commits.
4. Update documentation if necessary.
5. Ensure tests pass.
6. Open a Pull Request.

Please keep Pull Requests as small and focused as possible.

---

# Coding Standards

The reference implementation should be:

- readable
- modular
- well documented
- extensively tested

Business logic should remain independent from client applications.

---

# Reporting Bugs

Please include:

- HCP version
- environment
- expected behavior
- actual behavior
- reproduction steps

---

# Feature Requests

Before proposing a new feature, ask yourself:

Does this improve the Humanitarian Connection Protocol?

Or does it belong in a client application?

Protocol changes should only be proposed when they improve interoperability or the protocol itself.

---

By participating in this project you agree to follow our Code of Conduct.
